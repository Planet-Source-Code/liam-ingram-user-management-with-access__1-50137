Program Highlights < Table of Contents>

 

1. Icons.  Any menu item with the exception of separator bars can have an image.  Images can be from a control, an ImageList or memory.

 

2. Sidebars. Each menu panel can have its own sidebar.  The sidebar can consist of an image or user-defined formatted text.

 

3. Text on raised or sunken separator bars

 

4. Contents of a list box, file list, or combo box can be brought to a menu on the fly. In addition, if these menu items are clicked, the menus will trigger a click event in the associated control.  Multiselect file lists and list boxes are supported.

 

5. There are seven custom menus that can be called simply by referencing them in a caption. See Custom Menus for more information.

 

6. Optional menu highlighting provided. Selected menu items can be italicized, back highlighted using gradient colors vs. solid colors and disabled menu items can be prevented from being highlighted.  See the module properties section for more information.

 

7.  Menus can have tips which are displayed where you want and how you want.  See the cTips Class section for more information.

 

8.  The custom menu lvDrives can display system’s drives with their associated icons.

 

9.  If referencing a files list or list box containing file names, the icon associated with that file can be displayed along side the file name.

 

10. The required coded captions can be provided three different ways:

 

      - entered in the caption property of each menu item. Each flag is listed and described in a reference table.

      - created on the fly using the menu caption creation functions provided.

      - entered into a list box for easier management.  See the CACHE flag for more information.

 

11.  System menus are now handled by this version and a little color is brought to them also.

 

12. An optional user control “ctrlMenusSample” has been provided. This user control when added to a project in IDE will show most of the menus as they would look during run time.  There are restrictions.  Simply add the control to a form in Design view and double click the “HelpHook” property.

 

13.  Basically one function call does 99% of the work.  Other functions are provided for flexibility and optional control.

Get started… see Quick Start -- How-To.

 

 

Restrictions / Known Issues < Table of Contents>

 

The program has been tested on all systems with the exception of Windows NT v3.5

 

This program does not apply to the top level menu bar. It only applies to its menus and submenus.

 

SIDEBARS & CUSTOM MENUS

With versions of Windows ’98 and above, menus can scroll.  With previous versions of Windows, if a menu exceeded the height of the screen, the menus would appear in multiple columns.  This program can create colorful, informational, and enabled sidebars that can be used as advertising, hyperlinks, and/or executable links.  The sidebars, if used, are placed in their own column within a menu.  Windows does not support scrolling if multi-column menus are used.  Because of this, the program will separate menus into multiple columns in order to display the sidebar, if needed.  Should the menu items exceed 100+ in count, the multi-column menu could actually be so wide as to not fit the entire menu on the width of the screen.  If you foresee menu items exceeding 100 items, suggest you include the optional sidebar flag of NOSCROLL.  This will prevent the sidebar from being displayed, thereby negating multi-column menus, and allow the menus to scroll as designed.

 

If a menu item opens another menu (i.e., has submenus) or is a sidebar then the following restrictions also apply.

- These menu items cannot reference list box, combo box, file lists or custom menus. Their submenu items can.

- These menu items will not display shortcut/hot keys; although they can still be used

- Sidebars, if used, must always be the first menu item in the menu group or panel.  If a sidebar is encountered that is not the first item, it will be converted to a standard separator bar.

 

Custom menus do not support sidebars at this time.

 

Please read the expected system performance degradation if you choose to use the lvFonts custom menu.

 

API GENERATED MENUS

If you create your own menus via the various APIs and format them for LaVolpe SubMenus v2.0, keep the following restrictions in mind.

- Always use a unique menu ID for each menu item created.  Additionally, do not use menu IDs in the range of 2500-3500; these are used by this program

- Menu IDs can be duplicated as long as the menu items are not on the same submenu. This is because the menu items are stored by both their submenu and menu item IDs.

- If calling a formatted menu item via the TrackPopupMenu API, do NOT use the TPM_NONOTIFY flag.  Doing so will prevent menu items being measured and drawn

 

MENU SHORTCUT S

The program does not trap for shortcuts. If you provide a shortcut via the VB Menu Editor you should have no problems. However, if you provide shortcuts with the flag HOTKEY:, then you must trap for that shortcut via your form’s KeyPreview property and Key_Down/Key_Press events

 

SUBCLASSING FORMS

This program subclasses any form that calls the SetMenu function and all children of any MDI form that calls the SetMenu function. If you need to subclass your form further, run tests as to whether subclassing prior to or after the call to SetMenu has any negative effects.  The following Windows messages are subclassed in the module modMenus. If your subclassing routine also traps these messages, ensure there are no conflicts between your routine and modMenus’ MenuMessages routine.

 

WM_ACTIVATE, WM_COMMAND, WM_DESTROY

WM_DRAWITEM, WM_ENTERIDLE, WM_ENTERMENULOOP
WM_EXITMENULOOP, WM_INITMENUPOPUP, WM_MEASUREITEM
WM_MENUCOMMAND, WM_MENUSELECT, WM_MDIACTIVATE

WM_MDIDESTROY, WM_MDIMAXIMIZE

 

Quick Start -- How-To < Table of Contents>

 

1. An image list isn’t absolutely necessary as images can be from other controls like Image or Picturebox controls that have a Picture property. Images can also come from memory variables/handles like the StdPicture. However, to get started, add an ImageList control to your form and add a few images to the control.

 

2. Using VB’s Menu Editor (Ctrl+E), add a few menu items

 

3.  After each of the menu captions, add the flag {IMG:I#}, where # is the image index from the ImageList control. And save the changes.

 

4. In the form’s Load Event, add the following statement, changing the ImageList control name as needed:  SetMenu Me.hWnd, ImageList1

 

5. Run the application

 

The program offers dozens of options to format captions, bring control contents to menus, font and highlighting options, menu tips, and also custom, ready-made menus.  Please read thru this help menu or study the sample project provided to become familiar with the program’s capabilities.

 

Toolbars < Table of Contents>

The program can modify toolbar menu items also. That is, the so-called ButtonMenus.  There is one restriction that you need to be aware of. Even though the program can add custom menus to toolbar menus, they will never fire.  The limitation is due to the way toolbars are handled.

 

The common control’s toolbar class creates the button menus on the fly using TrackPopupMenuEx API with a flag of TPM_ReturnCmd.  This combination makes it impossible to trap for a custom menu.  The program never knows what menu is clicked because of that TPM_ReturnCmd flag.  And that fact that the TrackPopupMenu API is used indicates the control only tracks the menu items it creates.  Well, custom menus are add-on menu items that the control has no knowledge of.  There is no work around.  The capability to still include custom menus on toolbar button menus has been left in the program, should the toolbar class next version(s) allow manipulation of the button menus.

 

In addition, the program is lightly designed to recognize custom toolbars and may be able to draw their menus also.  Should you use a custom menu and want LaVolpe Menus to manage the drawing, please take a look at the AddToolbarClass function.

 

The toolbars menus are drawn by using the SetMenu function.  You would simply provide the toolbar handle, cTips class if any, and ensure you set the optional IsToolbar parameter to True.

The cTips Class <Table of Contents>

 

This class has three puposes.

 

#1. Return menu tips to the form that owns the menu. For MDI child forms, their cTips class is their parent’s cTips class by default. A child form can have its own initialized cTips class and using the RerouteTips function can have its menu’s tips passed to that child form.

 

#2. Return custom menu codes, Ids and values when user click on these menus.  See the various custom menus you can use.  These menus are ready-made and can be called as a popup menu using the PopupMenuCustom function or being attached to an existing menu item by using the Create function for the custom menu you want to display.

 

#3. Return keystrokes to a parent MDI form that has no children loaded.  The modMenus ReturnMDIKeystrokes property must be set to True.  Note: The function that returns the state of the Alt, Control and Shift keys do not work on WinME as that function has been disabled by Microsoft for WinME.

 

To use the cTips class, follow these steps.  Replace the variable name myTips with whatever you decide to use.

In the form where menu tips will be displayed and/or custom menu values will be processed… You can copy & paste the green lines below right into your project.

 

1. Declare the cTips class in the declarations section of the form. You can declare it as public if needed.

      Private WithEvents myTips As cTips

 

2. In the form’s Load Event you must initialize the class, otherwise menu tips will never be forwarded. Then pass the class to the SetMenu function.

      Set myTips = New cTips

      SetMenu Me.hWnd, [ImageList], myTips

 

3. In the form’s Unload Event you must terminate the class to release allocated memory.

      Set myTips = Nothing

 

4. Now you process the tips via the new events created by your declaration in step 1 above. You will have four new events:

 

     Private Sub myTips_DisplayTip(TipText As String)

      - within this event, simply display the tip wherever and however you choose. After all menus have been closed, a null tip will be passed.

 

     Private Sub myTips_CustomSelection(UserID As String, Category As String, Value As Variant)

      - within this event build a Select Case statement to trap the different types of Categories passed:

      Select Case Category

            Case “Color”                ‘ used with the lvColors custom menu, return value is Long (color value)

            Case “DayOfMonth”     ‘ used with  the lvMonth custom menu, return value is Date (date selected)

            Case “Drive”                ‘ used with the lvDrives custom menu, return value is String (drive:\)

            Case “Font”                  ‘ used with the lvFonts custom menu, return value is String (font name)

            Case “Month”               ‘ used with the lvMonths custom menu, return value is Long (month 1-12)

            Case “State”                 ‘ used with the lvStates custom menu, return value is String (string: FL, CA, DC, etc)

            Case “WeekDay”         ‘ used with the lvDays custom menu, return value is Long (weekday 1-7)

            Case “MenusClosed”    ‘ notifies you that the current menu loop as ended. You can use this notification to update your status bar, controls, etc.

      End Select

      - the UserID string is a custom value you provided when you created the menu captions. This is for your use as you see fit.

      - the value returned is described in each of the above functions

 

      Private Sub myTips_MDIKeyUp(KeyCode As Integer, Shift As Integer)

 

      Private Sub myTips_MDIKeyDown(KeyCode As Integer, Shift As Integer)

Notes:

1. When an IMG:, LB:, or CB: flag’s parameter is a form control, the control must exist on the same form that hosts the menu.  If the control is indexed, supply the index in parentheses after the control name ( i.e., imgIcon(8) ). If the parameter is a handle (hWnd), then the control can exist on any loaded form. Also see note 7 below.

 

2. The IMG: flag can reference an image three different ways. Also see note 7 below.

    - Handle in long or hexadecimal value. Hexadecimal values are preceded by the characters &H. (i.e., &HFF01)

    - If from an ImageList control, the reference is preceded by the letter I for ImageList.  (i.e., I5)

    - If from an image-type control with a Picture property, reference by control name (i.e., Picture1). See note 1 for indexed controls.

 

3. If a shortcut or hot key is provided with the HOTKEY: flag, you need to remember two things.

    - The parameter supplied will overwrite a hot key entered in VB Menu Editor for that menu item when the hotkey is displayed.

    - You need to trap for that hot key--the program does not.  Refer to form’s KeyPreview & KeyDown event in VB.

 

4. Scrolling menus in Windows cannot support multicolumn menus. Sidebars are created in their own column in order to make them “clickable”.  Because of this, any time a menu would normally scroll, that menu is converted into columns similar to Windows ’95.  The NOSCROLL flag will hide the sidebar on scrolling menus only, if you do not want the menus split up in columns.  However, if the sidebar is hidden, it cannot be clicked either.

 

5. Tips are displayed how and where you decide. Tips should always be the last flag in a caption. You must reference the cTips class in the form that will display/handle the tips.

 

6. Cached menu captions may be easier to manage vs. the long caption names entered in VB’s Menu Editor.  Since these are maintained in a list box, you can easily copy and paste captions to/from the List property of the list box.  For Cached captions the coded part of the caption can exist in one list box that is available at all times. In order to use this option, you must set the modMenus property MenuCaptionListBox to the hWnd of the list box prior to any menus being displayed. Examples would be:  File &Open{Cache:12}. In addition the extremely long caption for a text sidebar can be simply replaced with {Cache:#} where the list box item # would contain the entire coded caption for the sidebar. Also see note 7 below.

 

7. While in design mode and using the optional UserControl to view menus, cached captions, images, and menus containing list box/combo box items will not be displayed as they would in run-time. This is because an instance to the controls containing these items has not been created while in design mode and the coded menu items cannot be processed completely.  When in run-time, these issues do not apply.

 

8. The FILES: flag will return the icon associated with the files contained in the referenced list box.  If the list box contains the full path in the file name then provide a value of “-1” for this flag, otherwise, provide the full path of the files contained in that list box.  This is the only flag that provides tips automatically.  The tips will be the file description which is beneficial for .EXE type files.

Troubleshooting  <Table of Contents>

 

Following could be common problems experienced with LaVolpe Submenus v2.  Most errors/problems are probably related to improperly coded menu captions and can be resolved by using the proper Create functions.

 

Error: Image sidebar was changed to "La Volpe Menus"

Reason: You supplied an image for your sidebar, but that image wasn't found.

Fix: Ensure the image you selected exists or if the image is from a control, ensure that the control has a "Picture" property and that the control is on the same form as the menu.

 

Error: Sidebar text did not display vertically

Reason: Font chosen does not support vertical drawing or the font name you provided doesn't exist and Windows was not able  to supply a close match, usually a typo.

Fix: Use only common TrueType fonts expected to be in place on any system (i.e., Arial, Times New Roman, Tahoma, etc), or  build a routine to ensure the font you want to use is installed, or install the font via your application's setup routines

 

Error: Sidebar width is too large

Reason: Sidebars are forced to be a minimum of 35 pixels. They cannot be less. Text sidebar widths can be automatically enlarged to fully display the text.

Fix: For text sidebars: Supply a smaller font in either the FSize: and/or MinFSize: flags.

 

Error: Text separator bars are not sizing properly

Reason: Win98 or WinME issues when the submenu only consists of separator bars

Fix: Do not toggle fonts while application is running. If offering user a font choice, save that choice, inform user that font change will take place when application next starts and then set the modMenus.MenuFontName and modMenus.MenuFontSize properties in your primary startup routine before the first call to SetMenu

 

Error: List box, combo box or files list contents are not being displayed on a menu as expected.

Reasons:

        1. The control you referenced with the LB: or CB: flag does not exist on the same form as the menu or the wrong flag was used for the wrong type of control.

        2. You referenced the control on a menu item that opens a submenu

        3. If the control is a user-drawn control, then this program cannot support it.

Fix:

        1. Ensure the control exists on same form as the menu displaying the contents of the control and that CB: flag is used for combo boxes and the LB: flag is used for list boxes & file lists.

        2. The contents of list boxes and combo boxes are displayed on a submenu associated with the menu item referencing it. These controls will only work on menu items that do not open submenus. Otherwise, the contents of the control would replace the existing submenu without any warning and this program will not allow that.

        3. None.

 

Error: A custom menu is not displaying.

Reason: You referenced the custom menu on a menu item that opens a submenu or the caption is incorrectly coded.

Fix: Custom menus are displayed on a submenu associated with the menu item referencing it. These custom menus will only work on menu items that do not open submenus. Otherwise, the contents of the custom menu would replace the existing submenu without any warning.

 

Error: Menu icons stopped displaying on the menus

Reason: The image list control you referenced when you called the SetMenu function no longer exists because the form containing that control has closed or the control containing the image (i.e., Image1, Picture1, etc) is not available for the same reason.

Fix: Place the image list control on the same form as the menu or on another form that will be open all the time. Don't forget to reference the image list correctly in the call to the SetMenu function

 

Error: Icons displayed with file names are not the same as those used in my Explorer Window.

Reason: The icons chosen in the program are from the program that opens the file name in question. This is not always the same as the ones displayed in Windows Explorer. The reason is that the function used to extract the same icons used in Explorer must reduce the default image from 32 pixels to 16 pixels and loss of quality is evident. Instead, the program queries the associated application's 16 pixel icon library and displays the first one returned. The quality is excellent.

Fix: None

 

Error: Cached captions are not showing up

Reason: The list box you referenced in the modMenus.MenuCaptionListBox is no longer available because the form closed.

Fix: Place the list box on a form that is always open. And if not possible, then do not use the CACHE: flag and provide the coded captions with the actual menu captions

 

Error: The coded captions are being displayed on the menu item and shouldn't be

Reason: The coded caption is not surrounded by brackets { }. Or the form did not make a call to the SetMenu function.

Fix:  Ensure the { } symbols are used vs the [ ] or ( ), and place a call to the SetMenu function in the form’s Load event.

 

Error: Only some of the coded captions seem to be taking effect.

Reason: Most likely the entire caption contains more than one set of { } brackets

Fix: Ensure only one set of { } symbols are used in any menu caption. Additionally, try moving the Tip: flag as the last flag if appropriate.

 

Error: The menu tips are not being displayed

Reason:

        1. The cTips class was not intialized in your form

        2. The cTips class has been rerouted to another form

        3. The tips are from a toolbar. When the SetMenu function was called, passing the toolbar hWnd, no cTips class was passed.

Fix:

        1. Ensure you initialize the class by  Set myTips = New cTips

        2. You have rerouted the cTips via the RerouteTips function and did not route it back or you routed the tips via the PopupMenuCustom and expected the routing to stay. That function reroutes tips (if parameter provided) while the popup menu is active and the routes it back to the previous setting when the popup menu closes.

        3. Although MDI child forms are automatically subclassed using the its parent’s settings, this is not the case for toolbars.  Toolbars, if using tips, must supply the cTips class in the SetMenu function call.
